#### 回溯

回溯可以很简单地做这道题，但是时间复杂度会很高

```c++
class Solution {
public:
    int count = 0;

    void backtrack(vector<int>& nums , int target, int index, int sum){
        if (index == nums.size()) {
            if (sum == target) {
                count++;
            }
        } else {
            backtrack(nums, target, index + 1, sum + nums[index]);
            backtrack(nums, target, index + 1, sum - nums[index]);
        }



        // 也可以写成显式的回溯
        // sum  = sum + nums[index];
        // index++;
        // backtrack(nums, target, index, sum);
        // index--;
        // sum = sum - nums[index];
        
    }


    int findTargetSumWays(vector<int>& nums, int target) {
        backtrack(nums, target, 0, 0);
        return count;
    }
};
```

#### 动态规划

如果直接根据这个问题来分析，可以得出`dp[i][j] = dp[i - 1][j - num] + dp[i - 1][j + num]`，但是存在问题：i是i之前的元素，j是和，但是由于可以取负，所以j的范围里面也有负数，而dp的下标不能为负

------

**所以需要转换一下问题**

假设数组中所有数字的总和为 `sum`。

我们将数组中的数字分为两组：

- **$P$ (Positive)**：前面加 `+` 的数字之和。
- **$N$ (Negative)**：前面加 `-` 的数字之和。

根据题目要求，我们要满足：

1. $P - N = \text{target}$
2. $P + N = \text{sum}$

将两个等式相加：

$2P = \text{target} + \text{sum}$

由此得出：

**$P = \frac{\text{target} + \text{sum}}{2}$**

**结论**：问题转化成了——**在数组中找到若干个数字，使得它们的和恰好等于 $P$。** 共有多少种组合方式？这正是 0-1 背包的变体。

```c++
class Solution {
public:
    int findTargetSumWays(vector<int>& nums, int target) {
        int n = nums.size();
        int sum = 0;
        for(int num : nums){
            sum = sum + num;
        }
        //如果target的绝对值比sum大，那么无论全为正还是全为负，都不可能凑出target
        //如果target + sum等于奇数，那么他们除以2就不是整数，也不可能由全是整数的nums凑出来
        if(abs(target) > sum || (target + sum)%2 != 0) return 0;

        //题目转换为找到若干元素加起来等于positive，直接忽略掉了负数的处理，让题目更简单了
        int positive = (target + sum) / 2;
	
        //dp[i][j]表示在前i个元素中有多少种方法可以凑出i，i的范围0-n，j范围0-positive
        vector<vector<int>> dp(n + 1, vector<int>(positive + 1, 0));

        dp[0][0] = 1;

        for(int i = 1; i<=n; i++){
            for(int j = 0; j <=positive; j++){
                //dp[i][j]的方案数等于取i和不取i两种方案之和，如果j比nums[i -1]小的话，那就只有第一种方案
                dp[i][j] = dp[i - 1][j];
                if(j >= nums[i-1]){
                    dp[i][j] += dp[i - 1][j - nums[i - 1]];
                }
            }
        }

        return dp[n][positive];
    }
};
```

#### 动态规划的一维简化

> [!WARNING]
>
> 一维简化只是看起来是一维，实际上他的迭代关系，底层逻辑和二维是一样的

- 初始状态下，dp[j]相当于都是`dp[0][j]`，也就是都在第一行，外层的循环每经过一轮，i就会隐式的变大，如果此时访问一次dp[j]，dp[j]就会往下面挪
- 内层循环的顺序问题，如果二维的内层循环是从小到大，那么一维的就是从大到小。比如`dp[i][j] = dp[i - 1][j - num]`，对于二维来讲，j依赖j - num，所以要先算j-num，就是从小到大；但是在一维中，j-num必须是旧数据，即上层的数据，所以要先更新右边，从大到小

```c++
class Solution {
public:
    int findTargetSumWays(vector<int>& nums, int target) {
        int n = nums.size();
        int sum = 0;
        for(int num : nums){
            sum = sum + num;
        }
        //如果target的绝对值比sum大，那么无论全为正还是全为负，都不可能凑出target
        //如果target + sum等于奇数，那么他们除以2就不是整数，也不可能由全是整数的nums凑出来
        if(abs(target) > sum || (target + sum)%2 != 0) return 0;

        //题目转换为找到若干元素加起来等于positive，直接忽略掉了负数的处理，让题目更简单了
        int positive = (target + sum) / 2;

        vector<int> dp(positive + 1, 0);

        //此时，i还在0上面，所以这个其实等效于dp[0][0] = 1
        dp[0] = 1;

        for(int num : nums){
            //j必须从大到小，因为dp[j]需要用到dp[j - num]，而只有左边的是上一行的旧数据时，这个公式才是对的，所以要保证左边是旧的，右边是新的，从右边开始更新，这个顺序和二维dp是相反的
            for(int j = positive; j >= 0; j--){
                //这一行看起来屁用没有，实际上也是屁用没有，但是他代表着从上一行挪了下来，即dp[i][j] = dp[i - 1][j]
                dp[j] = dp[j];
                if(j >= num){
                    dp[j] += dp[j - num];
                }
            }
        }

        return dp[positive];
    }
};
```
